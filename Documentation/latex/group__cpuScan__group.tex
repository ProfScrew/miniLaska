Cpu that goes through all combinations and at the end scans boards. 

This function checks available movement. If there is none the its game lost, if one move available it plays it without calling the recursive function.

But if there is more then one available movement it calls the revursive function for each of available movement that calculates the value of the movement.

After its done it picks the move with the highest value.

Recursive function for the Cpu Scan. 

Goes through every possible move to a certain depth and scans the end cases.

Cases\+:
\begin{DoxyItemize}
\item If no possible movement it registeres that the game is lost/won and saves value.
\item If depth is zero it scans the board and gets a value.
\item If the depth is not zero it checks the moves and recals the \mbox{\hyperlink{group__cpuScan__group_ga23e967ede87ba6c5e161e48fbb9df68e}{cpu\+Scan\+Rec}} and decreases depth.
\end{DoxyItemize}

At the end the function returns the value of the move.

\begin{DoxyNote}{Note}
The function also keeps track of the number of outcomes so after its done, the resault is devided by the outcomes So the cpu doesn\textquotesingle{}t prioritize the longest game but the more advantagious.
\end{DoxyNote}
