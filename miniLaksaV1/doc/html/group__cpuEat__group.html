Cpu that goes through all combinations and takes count of the captures.

Cpu Eat function that focuses on capture to evaluete the best move.

Checks available movement. If There is none then its game lost, if only move available it plays it without calling the recursive function.

But if there is more then one available movement it calls the recursive function(cpuEatRec) for each of available movement that calculates the value of the movement.

After its done it picks the move with the highest value.

Parameters
tempBoard where game is played.
depthDepth used by recursive function.

Recursive function for the Cpu Eat.

Goes throught each of possible move to a certain depth and takes in consideration the captures

Cases:

  • If no available movement it registeres the value as a win/loss and then devides by the depth count
  • If depth is zero then the value is devided by the depth count and returned
  • If depth is not zero it checks move and for each move it calls the function #cpuEatMove:
    • If it's a normal move it does nothing
    • If capture move then it adds or subtracts a certain value
Parameters
originalBoard on which the function is working on.
depthKeeps track of how much still has to go into depth
depthCountKeeps track of current depth and used to divide the value at the end so a win of a deeper depth has less value then the one higher
valueStores the value of the move, each time there is a capture it is increased or decreased depending on the side
numberOfOutcomesCounts the possible outcomes it is used later in the cpuEat to devide the returned value
sideTracks the side, used in the value manipulation
